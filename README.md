As asked by the Course Leader

Following a discussion with Saad on 28/04/26, it was identified that limited knowledge of using GitHub correctly had affected the group’s workflow. As a result, the group continued the project over several weeks without consistently uploading or committing work. An initial attempt was made to use the repository; however, this led to confusion, with some members unable to view uploaded files and issues connecting through Visual Studio.
Due to these difficulties, the group decided to delete the original repository, as it had become disorganised and difficult to manage. The focus then shifted to completing the task to the best possible standard by sharing code and report sections directly between members.
On 27/04/26, a new GitHub repository was created to manually upload individual contributions. This allowed each group member’s work to still be evidenced. Although a full commit history was not available, a record of progress was maintained through a project planner, including meeting dates, locations, tasks completed, and time spent on each section.
